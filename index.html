How I Built My Portfolio from Scratch

Published: July 5, 2025   |   Category: Web Development

1. Planning & Goal Setting

I started by outlining my goals: showcase my web development, design, and security skills, and make it easy for potential clients or employers to contact me. I sketched a rough layout and listed the sections I wanted: Hero, About, Projects, Blog, and Contact.

2. Design & Inspiration

I chose a clean, modern look with a white background, blue and black accents, and the Inter font for readability. I used Figma to design the layout, focusing on simplicity, clear navigation, and strong visual hierarchy. I also made sure the design would look great on both desktop and mobile devices.

3. Development

I built the site using HTML, CSS, and JavaScript. I structured the HTML semantically, used CSS Grid and Flexbox for layout, and added smooth animations for a polished feel. I implemented project filtering, a functional contact form, and modal image viewing for design work.

4. Making It Responsive

Responsiveness was a top priority. I used media queries to adjust layouts, font sizes, and spacing for tablets and phones. Images and cards scale fluidly, and the navigation switches to a hamburger menu on small screens.

5. Performance & Optimization

I optimized images, enabled lazy loading, and minified CSS/JS for fast load times. I also used font-display: swap for fonts and ensured accessibility best practices.

6. Deployment

Once everything was tested, I deployed the site using GitHub Pages for free, fast hosting. I set up a custom domain and shared the site with my network.

Conclusion

Building my portfolio from scratch was a rewarding experience. It not only showcases my skills but also serves as a living project I can update as I grow. If you have questions or want tips for your own portfolio, feel free to reach out!
